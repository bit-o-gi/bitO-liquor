# Core Beliefs

## 제품
- 사용자는 "예쁜 카드"보다 "믿을 수 있는 가격 비교"를 먼저 원한다.
- 카탈로그는 탐색 진입점이고, 상세 페이지는 비교를 이어가는 보강 경로다.
- 디자인은 정보 밀도를 낮추기보다, 높은 정보 밀도를 덜 피곤하게 보여주는 쪽으로 설계한다.

## 아키텍처
- 읽기 경로는 UI에 가깝게 둔다.
- 쓰기 경로는 크롤러에 집중시킨다.
- 데이터 모델 변화는 프론트 UI보다 먼저 계약 문서에 반영한다.
- 설계 문서는 이슈가 아니라 아티팩트 기준으로 유지한다.

## 실행
- 진행 중 작업은 `docs/exec-plans/active/`에 둔다.
- 완료된 작업은 `docs/exec-plans/completed/`로 이동한다.
- 장기 가치가 남는 결정만 `product-specs/`, `design-docs/`, 루트 문서로 승격한다.
- 검증 없이 완료를 주장하지 않는다.
