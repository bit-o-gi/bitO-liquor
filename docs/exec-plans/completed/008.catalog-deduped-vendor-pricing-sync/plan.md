## 현재 구조 메모
- 기존 `liquor_catalog_latest_price` 뷰 정의는 `distinct on (lp.liquor_id)` 기반이라 주류당 최신 가격 1행만 반환한다.
- 따라서 DB가 이미 중복 주류를 제거한 상태라면, 이 뷰를 그대로 쓰는 순간 멀티벤더 정보는 구조적으로 사라진다.
- 현재 이 이슈에서는 `liquor`를 페이지네이션 기준으로 조회하고, 후속으로 `liquor_price`를 `liquor_id in (...)`으로 읽어 같은 주류의 판매처 가격들을 서버에서 집계하는 방향으로 정리했다.
- 서버는 `liquor_id + source` 기준 최신 가격만 남기고, 그 결과를 카드 단위 `vendors[]`와 `lowest_price`로 내려준다.
- 프론트는 더 이상 `name` 기준 재그룹핑을 하지 않고, 서버가 내려준 카드 모델을 그대로 렌더링한다.
- 프론트 검증 결과 `npm run lint`, `npm run test`, `npm run build`가 모두 통과했다. 단, Codex 샌드박스 안의 `next build`는 Turbopack CSS 처리 과정에서 포트 바인딩 제한으로 실패할 수 있어 실제 빌드 확인은 샌드박스 밖 실행 결과를 기준으로 기록한다.
