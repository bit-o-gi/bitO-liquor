# Brief

## 문제
- 현재 저장소에는 repo harness와 application harness 문서가 있지만, 이 둘을 함께 지배하는 단일 최상위 규범 문서는 없다.
- 그래서 어떤 원칙이 절대적이고, 어떤 문서가 그 원칙을 구체화하는지 문서 우선순위가 아직 분산되어 있다.

## 목표
- 저장소 전체를 지배하는 `CONSTITUTION.md` 초안을 만든다.
- repo harness와 application harness가 이 헌법 아래에 놓이도록 상위 문서를 연결한다.

## 비목표
- 기존 제품 계약과 설계 세부를 다시 설계하지 않는다.
- 헌법을 장황한 설계 문서로 만들지 않는다.

## 성공 기준
- 루트에 `CONSTITUTION.md`가 추가된다.
- `AGENTS.md`, `README.md`, `docs/REPOSITORY.md`, `ARCHITECTURE.md`가 헌법을 참조한다.
- 헌법이 문서 우선순위, 불변 원칙, 개정 규칙을 명확히 담는다.
