# Lean PRD

## 문제
현재 메인 상단 헤더의 `Jururuk` 로고 영역은 브랜드 식별성이 약하고, 긴 카탈로그를 스크롤한 뒤 빠르게 최상단으로 돌아가는 진입점 역할도 충분히 하지 못한다.

이 변경을 별도 이슈로 분리하지 않으면 단순 UI 보강과 카탈로그 기능 변경 이력이 섞여 추적이 어려워진다.

## 목표
- 메인 상단 헤더의 로고 영역에 프로젝트 이미지를 추가한다.
- 이미지와 `Jururuk` 텍스트를 함께 클릭했을 때 페이지 최상단으로 부드럽게 이동하도록 한다.
- 기존 카탈로그 조회, 검색, 무한 스크롤 동작은 유지한다.

## 비목표
- 헤더 전체 레이아웃을 전면 개편하지 않는다.
- 브랜딩 리뉴얼 범위로 확장하지 않는다.
- 검색 상태나 데이터 요청 흐름을 변경하지 않는다.

## 사용자 가치
- 브랜드 인지가 더 빨라진다.
- 긴 목록 탐색 중에도 상단 복귀가 쉬워진다.
- 작은 상호작용 개선을 별도 이슈로 관리할 수 있다.

## 핵심 요구사항
- 헤더 로고 영역은 이미지와 `Jururuk` 텍스트를 함께 보여야 한다.
- 로고 버튼 클릭 시 최상단으로 스무스 스크롤되어야 한다.
- 클릭 동작은 검색 상태를 불필요하게 초기화하지 않아야 한다.
- 작업 범위와 진행 상태는 `005` 문서에 기록한다.

## 성공 기준
- 헤더 좌측 브랜드 영역에 이미지가 노출된다.
- 스크롤을 내린 뒤 로고 클릭 시 페이지가 최상단으로 이동한다.
- 검색 입력, 목록 조회, 무한 스크롤에 회귀가 없다.

## 범위
포함:
- 헤더 로고 UI 보강
- 상단 복귀 인터랙션 추가
- 관련 문서화와 기본 검증

제외:
- 신규 페이지 추가
- 데이터 계층 수정
- 전체 내비게이션 구조 변경

## 제약 및 전제
- 기존 카탈로그 헤더 구조 안에서 해결한다.
- 로고 이미지는 프로젝트에 포함 가능한 정적 자산 형태로 제공한다.
