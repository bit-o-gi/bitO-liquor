# Lean PRD

## 문제
현재 카탈로그 카드의 판매처 오버레이는 외부 판매처 링크만 제공하고 있어, 사용자가 상품 자체의 상세 정보를 확인할 수 있는 내부 진입 경로가 없다.

특히 데스크톱에서는 카드 hover 오버레이가 핵심 인터랙션인데, 여기서 상세 페이지로 이동할 수 없으면 상세 탐색 흐름이 끊긴다.

## 목표
- 주류 상세 페이지를 새로 만든다.
- 카탈로그 카드의 오버레이에서 상세 페이지로 이동할 수 있게 한다.
- 판매처 외부 링크와 상세 진입이 충돌하지 않도록 인터랙션을 정리한다.

## 비목표
- 상세 페이지에서 가격 히스토리 차트나 리뷰 기능을 추가하지 않는다.
- 상세 페이지 전용 복잡한 추천 알고리즘을 도입하지 않는다.
- 백엔드 Spring API를 새로 추가하지 않는다.

## 사용자 가치
- 카탈로그에서 관심 상품을 발견한 뒤 바로 내부 상세 화면으로 이동할 수 있다.
- 판매처 가격 비교와 상품 기본 정보 탐색 흐름이 자연스럽게 이어진다.

## 핵심 요구사항
- Next.js App Router 기준 상세 페이지 라우트를 추가한다.
- 상세 페이지는 최소한 상품명, 브랜드, 카테고리, 국가, 도수, 용량, 이미지, 판매처별 가격을 표시해야 한다.
- 상세 조회는 현재 Supabase 기반 데이터 구조와 일치해야 한다.
- 데스크톱 오버레이에서는 빈 영역 클릭으로 상세 페이지에 진입할 수 있어야 한다.
- 판매처 링크는 기존처럼 외부 페이지로 열려야 하며 상세 이동과 충돌하면 안 된다.
- 모바일에서도 상세 페이지로 이동할 수 있는 명시적 진입 버튼이 있어야 한다.

## 성공 기준
- 카탈로그 카드에서 상세 페이지로 진입할 수 있다.
- 상세 페이지가 현재 DB 구조 기준 상품과 판매처 가격을 정상 렌더링한다.
- 기본 검증 결과와 결정 사항이 문서에 남는다.

## 범위
포함:
- 상세 페이지 라우트 추가
- 상세 조회 서버 로직 추가
- 카탈로그 카드 상세 진입 인터랙션 추가
- 관련 문서 갱신

제외:
- 신규 상세 API 서버 분리
- 상세 페이지 SEO 고도화
- 가격 이력 차트, 즐겨찾기, 리뷰

## 제약 및 전제
- 현재 `frontend`의 Supabase 조회 경로를 그대로 활용한다.
- 상세 라우트 식별자는 우선 `liquor.id`를 사용한다.
- 이미 존재하는 카드 및 오버레이 시각 톤은 최대한 유지한다.
