## 방향
- 상세 페이지는 `frontend/app/liquors/[id]/page.tsx`에서 서버 컴포넌트로 렌더링한다.
- 카탈로그와 같은 Supabase 읽기 계층에서 단일 주류 + 판매처 가격 목록을 조회해 상세 모델로 정리한다.
- 카드 오버레이는 내부 상세 진입과 외부 판매처 링크를 동시에 지원하되, 링크 우선순위가 명확하도록 레이어를 분리한다.

## 현재 구조 메모
- 상세 라우트 식별자는 우선 `liquor.id`를 사용한다.
- 상세 페이지 조회는 카탈로그와 동일하게 `liquor` + `liquor_price`를 읽고, `source`별 최신 가격만 남긴다.
- 데스크톱 카드 오버레이는 절대 배치된 내부 링크 레이어로 상세 진입을 제공하고, 판매처 외부 링크는 상위 z-index로 유지해 클릭 충돌을 방지한다.
- 모바일은 hover 오버레이가 없으므로 카드 하단의 `상세 보기` 버튼을 별도로 제공한다.
