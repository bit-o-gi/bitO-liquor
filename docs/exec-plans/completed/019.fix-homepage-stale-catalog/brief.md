# 019.fix-homepage-stale-catalog / brief

## 배경
- 라이브 홈 화면의 첫 카드 목록이 `/api/liquors` 응답과 다르게 보였다.
- 확인 결과 홈 페이지가 빌드 시점 데이터로 정적 생성되어, 현재 API의 최신 카탈로그 데이터와 어긋날 수 있었다.

## 목표
- 홈 첫 화면이 현재 Supabase 기준 카탈로그 데이터와 일치하도록 만든다.
- `/api/liquors`와 홈 초기 렌더링의 데이터 불일치를 제거한다.

## 범위
- `frontend/app/page.tsx`
- 실행 기록 문서

## 비범위
- 카탈로그 조회 쿼리 로직 자체 변경
- 데이터 계약 변경
