# Implementation Plan

## 상태 규칙
- `[ ]` 미착수
- `[-]` 진행 중
- `[x]` 완료

## 방향
- 이번 이슈는 기능 추가가 아니라 UI 회귀 복구 작업이다.
- Next.js App Router, 내부 API route, Supabase 조회 구조는 유지하고 프론트 표현 계층만 조정한다.
- Git 히스토리의 `frontend/src/App.tsx`를 기존 경험의 기준점으로 삼되, 추천 기능 관련 흐름은 제외한다.
- 현재 정상 동작 중인 검색, 목록, 무한 스크롤을 깨뜨리지 않는 범위에서 단계적으로 복구한다.

### 기준 UI 요약
- 기준 커밋은 `4a45afa`의 `frontend/src/App.tsx`로 잡는다.
- 상단은 sticky 헤더 + 로고 버튼 + 우측 정렬 검색창 구조였다.
- 본문은 검색 전용 상태가 아닐 때 히어로/소개 성격의 상단 섹션이 먼저 나오고, 그 아래 카탈로그 그리드가 이어지는 흐름이었다.
- 카드에서는 브랜드/카테고리, 제품명, 국가·도수·용량, 최저가가 한 번에 보였고 hover 시 판매처별 가격 오버레이가 노출됐다.
- 전체 톤은 단순한 앱 셸보다 "카탈로그 랜딩"에 가까운 여백, gradient, 카드 depth 중심 구성이었다.

### 복구 대상
- `CatalogPageClient.tsx`의 sticky 헤더, 로고 인상, 검색창 배치 복구
- 검색 전 기본 상태에서 보이는 상단 소개/요약 섹션 복구
- 카탈로그 메타 정보와 검색 결과 카피 정리
- 카드의 정보 밀도, 가격 강조, hover overlay 인상 복구
- 전역 배경, 섹션 여백, 카드 주변 분위기 복구

### 제외 대상
- `setPage`, `PreferenceTest`, `FeaturedPick`, `TestLoadingPage`, `TestResultPage` 등 추천/테스트 흐름
- "나의 위스키 취향 찾기" CTA와 해당 상태 전이
- 추천 결과 fallback 생성 로직과 추천 전용 섹션

### 회귀 체크리스트
- 현재 헤더는 텍스트 중심으로 축약돼 로고 버튼형 인상이 사라져 있다.
- 현재 히어로는 기술 마이그레이션 안내 문구 위주라 기존 카탈로그 경험과 어긋난다.
- 현재 그리드는 검색 상태/기본 상태를 구분하는 맥락 정보가 약하다.
- 현재 카드는 정보 위계와 오버레이는 남아 있으나 depth, 배지, 메타 정보, hover 연출이 평면적이다.
- 현재 `globals.css`는 최소 배경만 있어 섹션/카드/입력창의 공통 톤을 지지하지 못한다.

### 완료 기준
- 어떤 요소를 얼마나 복구해야 하는지 문서만 봐도 판단 가능해야 한다.
- 추천 관련 영역을 제외한 복구 범위가 확정되어야 한다.
