## 방향
- 이번 이슈는 구조 변경이 아니라 디자인/레이아웃 마감 보정이다.
- 핵심 기능은 유지하고, 전역 배경과 섹션 레이어의 시각 일관성을 우선적으로 해결한다.
- 원인을 확인하기 전까지 추측성 스타일 수정은 최소화하고, 실제 렌더 기준으로 문제 레이어를 특정한다.

## 리스크
- 실제 원인을 특정하지 못한 상태에서 전역 스타일만 반복 수정하면 증상이 계속 남을 수 있다.
- 고정 배경과 섹션 배경이 겹치면 성능 또는 가독성 문제가 생길 수 있다.

## 현재 메모
- 전역 배경이 스크롤 중 사라지는 현상은 단일 색상 문제라기보다, 긴 페이지에서 전역 톤이 약해지고 본문이 평면적으로 보이던 레이아웃 인상 문제에 가까웠다.
- `body` 고정 배경과 `catalog-shell` 내부 분위기 레이어를 분리하고, 카탈로그 본문에 반투명 surface 패널을 추가해 긴 페이지에서도 화면 톤이 유지되도록 조정했다.
- `CatalogPageClient.tsx`에는 소개 패널과 상태 메트릭을 추가해 이번 작업이 "배경 버그 패치"가 아니라 디자인/레이아웃 정리로 읽히도록 맞췄다.
- 최종적으로는 레이아웃 추가 보정안을 되돌리고, 페이지 배경만 단색으로 고정하고 카드 톤은 중립 흰색 계열로 조정하는 방향을 채택했다.
