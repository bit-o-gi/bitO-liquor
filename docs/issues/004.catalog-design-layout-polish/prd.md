# Lean PRD

## 문제
현재 프론트 카탈로그는 기능과 기본 UI 회귀 복구는 끝났지만, 실제 화면을 다시 점검해보면 디자인과 레이아웃 완성도가 아직 불안정하다. 특히 전역 배경이 스크롤 구간에서 끊겨 보이거나, 특정 섹션 레이어와 전역 배경의 경계가 어색하게 드러나는 등 "동작은 하지만 마감이 덜 된 화면" 인상이 남아 있다.

이 문제를 별도 이슈로 정리하지 않으면 구조 개선 이슈와 UI 회귀 복구 이슈의 범위가 섞이고, 이후 어떤 변경이 단순 스타일 보정인지 레이아웃 수준 수정인지 추적하기 어려워진다.

## 목표
- 카탈로그 홈 화면의 전역 배경, 섹션 레이아웃, 여백, 카드 주변 분위기를 한 번 더 다듬는다.
- 스크롤 시 배경이 사라지거나 끊겨 보이는 회귀를 해결한다.
- 현재 검색, 목록 조회, 무한 스크롤 등 기능 동작은 유지한 채 시각 완성도를 높인다.

## 비목표
- 카탈로그 구조를 다시 리팩터링하지 않는다.
- 데이터 fetch 계층, 도메인 모델, Supabase 쿼리 구조를 변경하지 않는다.
- 새로운 기능이나 신규 화면을 추가하지 않는다.
- 전면 브랜딩 리뉴얼로 범위를 확장하지 않는다.

## 사용자 가치
- 스크롤과 탐색 중에도 화면 톤이 안정적으로 유지된다.
- 홈 카탈로그가 임시 복구본이 아니라 마감된 제품 화면처럼 보인다.
- 이후 UI 조정 작업이 구조 이슈와 분리된 상태로 관리된다.

## 핵심 요구사항
- 전역 배경은 스크롤 길이와 무관하게 일관되게 보여야 한다.
- `CatalogPageClient`, `LiquorGrid`, `LiquorCard`, `app/globals.css`를 중심으로 레이아웃/시각 톤을 조정한다.
- 검색, 목록, 무한 스크롤, 초기 서버 선조회 동작은 유지해야 한다.
- 작업 범위와 진행 상태는 `004` 문서에만 기록한다.

## 성공 기준
- 페이지를 끝까지 스크롤해도 배경이 비거나 끊겨 보이지 않는다.
- 화면 상단부터 카드 목록 끝까지 배경/섹션 톤이 자연스럽게 이어진다.
- 기존 기능 검증이 회귀 없이 유지된다.
- 변경 범위와 이유가 `004` 문서에 명확히 남는다.

## 범위
포함:
- 전역 배경 레이어 조정
- 카탈로그 레이아웃/여백/섹션 톤 보정
- 필요한 범위의 카드/그리드 시각 조정
- 관련 검증 및 문서화

제외:
- 클린 아키텍처 구조 변경
- 신규 기능 추가
- 데이터 계층 수정
- 추천/개인화 흐름 도입

## 제약 및 전제
- 현재 프론트 기본 구조는 유지한다.
- 디자인 수정은 감각적 보정보다 회귀 해결과 일관성 확보를 우선한다.
- 특정 컴포넌트의 배경이 문제인지, 전역 캔버스 배경이 문제인지 실제 렌더 기준으로 구분해서 수정해야 한다.
